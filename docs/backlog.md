# RunAir — 개발 예정 사항 (Backlog)

개발이 확정되거나 논의된 기능을 기록합니다.
우선순위(P1/P2/P3) 기준: P1 = 다음 스프린트, P2 = 근시일, P3 = 장기

---

## 인증 / 사용자

| 우선순위 | 항목 | 설명 |
|--------|------|------|
| P2 | 로그인 기능 | 소셜 로그인(Google 등) 또는 이메일 기반 인증 |
| P2 | 개인 디폴트 페이스 설정 | 로그인 사용자가 자신의 기준 페이스를 저장하고 최초 진입 시 자동 적용 |

> 현재 미로그인 상태의 디폴트 페이스는 7:00/km (하드코딩, `src/app/page.tsx > DEFAULT_PACE`).
> 로그인 도입 시 서버에 저장된 사용자 페이스를 우선 적용하고, 없을 경우 7:00/km 폴백.

---

## API / 인프라

| 우선순위 | 항목 | 설명 |
|--------|------|------|
| P2 | 에어코리아 API 서버 캐싱 | 같은 측정소 데이터를 10~30분간 메모리 캐시. 개발계정 일일 500회 제한 대응 |
| P2 | 에어코리아 운영계정 전환 | data.go.kr 운영계정 신청 시 트래픽 한도 대폭 확대. 실사용자 증가 시점에 처리 |
| P3 | API 호출 사용자 단위 제한 | 동일 사용자의 중복 호출 방지 (IP 또는 세션 기준 rate limit) |
